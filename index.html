Balancing Strategies

Routing Algorithms

Six configurable strategies to match your throughput, budget, and latency requirements. Switch strategies at runtime without redeployment.

cost-optimized

Routes to the cheapest model that meets minimum quality thresholds. Ideal for high-volume batch jobs and non-critical tasks. Falls back to premium tiers on degraded output.

Budget High Volume

lowest-latency

Always selects the provider with the lowest p50 response time based on rolling 30-second health checks. Designed for real-time user-facing interactions.

Real-Time UX Priority

round-robin

Distributes requests evenly across all healthy providers in sequence. Prevents rate-limit bottlenecks and ensures balanced API quota consumption.

Balanced Rate Limits

quality-first

Always selects the highest-rated model for the detected task type. Uses prompt classification (code, reasoning, creative) to match model strengths.

Premium Task-Aware

weighted

Traffic is split proportionally across providers using configurable weights. Supports gradual migrations, A/B testing, and fine-grained budget allocation.

A/B Testing Migration

failover

Maintains a strict priority order. Routes to the primary until it degrades past a configurable error-rate threshold, then automatically cascades to backup providers.

Reliability Auto-Heal
Integration Patterns

Common Usage Patterns

Ready-to-use configurations for the most frequent orchestration scenarios. Each pattern includes n8n workflow snippets and Supabase schema references.

iOS Shortcut → n8n → LLM

Mobile

Trigger a prompt from iOS Shortcuts via webhook. The n8n workflow selects the model, invokes it, and returns the response to the Shortcut. 

// n8n webhook node config
{
  "method": "POST",
  "path":   "/prompt",
  "body":  {
    "prompt":   "{{$json.text}}",
    "strategy": "cost-optimized",
    "max_cost": 0.05
  }
}

Cascading Fallback Chain

Reliability

Automatically cascade through providers on failure or rate-limit. Each hop is logged to Supabase with error code and retry count. 

// Fallback priority chain
chain: [
  { model: "claude-sonnet-4" },
  { model: "gpt-4o",
    on_error: "next" },
  { model: "gemini-2.5-pro",
    on_error: "next" },
  { model: "ollama/llama3",
    on_error: "fail"  }
]

Task-Type Classification Router

Smart Routing

Classifies incoming prompts by task type and routes to the model best suited for that category — code, reasoning, creative, or summarization. 

// Task-type routing map
routes: {
  code:       "deepseek-coder",
  reasoning:  "claude-opus-4",
  creative:   "claude-sonnet-4",
  summary:    "gpt-4o-mini",
  default:    "gpt-4o"
}

Daily Budget Guard

Cost Control

Supabase tracks cumulative spend per workspace. When the daily budget is reached, requests automatically downgrade to local/free-tier models. 

-- Supabase cost guard query
SELECT sum(cost_usd)
FROM   llm_requests
WHERE  workspace_id = $1
  AND  created_at >=
       date_trunc('day', now());
-- > if sum > budget: fallback
